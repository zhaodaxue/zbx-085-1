# 街道绿化花箱浇灌计划 - 使用流程说明

## 一、启动应用

```bash
docker-compose up -d --build
```

启动后访问：
- 前端页面: http://localhost:5173
- 后端 API: http://localhost:3001

## 二、功能说明

### 花箱档案字段
| 字段 | 说明 |
|------|------|
| box_id | 花箱编号（唯一） |
| intersection_name | 路口名称 |
| last_humidity | 土壤湿度上次读数（0-100） |
| water_interval_days | 建议浇灌间隔天数 |
| last_water_date | 上次浇灌日期 |

### 核心操作
1. **登记浇灌**: 点击「登记浇灌」按钮，刷新上次浇灌日期为今天，湿度重置为 80%
2. **批量导入湿度**: 上传 CSV 文件，格式为 `box_id,humidity,read_date`
3. **优先级排序**: 列表按优先级从高到低自动排序

### 优先级规则
- **高优先级（红色标签）**: 距上次浇灌天数超过建议间隔 **且** 湿度低于 30%
- **中优先级（橙色标签）**: 满足上述任一条件
- **正常（无标签）**: 均不满足

优先级计算公式：
- 超期：100 + 超期天数 × 10 分
- 低湿度：100 + (30 - 湿度) × 2 分
- 总分越高，排序越靠前

## 三、示例流程：导入低湿度 → 列表置顶 → 登记浇灌后降级

### 第 1 步：查看初始状态
打开 http://localhost:5173
- 初始有 8 个示例花箱
- BX002（中山公园门口）湿度 28%，排在前列
- BX005（文化广场南侧）湿度 22% 且超期，排在第一位

### 第 2 步：导入低湿度 CSV
1. 点击「📥 导入湿度数据」按钮
2. 选择 `samples/humidity_import.csv` 文件（或拖拽上传）
3. CSV 内容如下：
```
box_id,humidity,read_date
BX001,25,2026-06-19
BX003,18,2026-06-19
BX006,42,2026-06-19
BX008,15,2026-06-19
```
4. 导入成功后，查看列表变化：
   - BX003（火车站北出口）: 湿度 18% + 超期 → **升置顶**
   - BX008（科技园大门）: 湿度 15% → **大幅前移**
   - BX001（人民广场路口）: 湿度 25% → **进入高优先级区**

### 第 3 步：登记浇灌（优先级降级）
1. 找到列表顶部的 BX003（火车站北出口）
2. 点击「💧 登记浇灌」按钮
3. 观察变化：
   - 上次浇灌日期变为今天
   - 湿度重置为 80%
   - 「超期」和「低湿度」标签消失
   - 卡片从红色变为白色
   - 列表中该花箱 **自动降级后移**

### 第 4 步：批量处理
依次对所有高优先级花箱执行「登记浇灌」，观察列表逐渐恢复正常排序。

## 四、API 接口

### 获取花箱列表（已排序）
```
GET /api/boxes
```

### 新增花箱
```
POST /api/boxes
Content-Type: application/json

{
  "box_id": "BX009",
  "intersection_name": "新建路口",
  "last_humidity": 50,
  "water_interval_days": 3,
  "last_water_date": "2026-06-19"
}
```

### 登记浇灌
```
POST /api/boxes/:id/water
```

### 批量导入湿度
```
POST /api/import/humidity
Content-Type: multipart/form-data

file: <CSV文件>
```

## 五、数据持久化

- SQLite 数据库文件位于 `backend/data/planter.db`
- Docker volume 挂载，容器重建后数据不丢失
